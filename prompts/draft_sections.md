# Draft Sections Prompt

## Task

Draft response content for each mapped template section, drawing on boilerplate material and case studies where appropriate.

## Instructions

1. For each template section with mapped requirements:
   - Review the requirement(s) mapped to that section.
   - Pull relevant content from the boilerplate library (see `boilerplate/`).
   - Reference applicable case studies (see `case_studies/`) to substantiate claims.
   - Draft a response that directly and compliantly addresses each requirement.
2. Tailor generic boilerplate language to the specific tender context, agency, and evaluation criteria.
3. Keep sections concise — prioritise quality over length.
4. Use active voice and quantify claims wherever possible (e.g., "delivered X in Y weeks with Z outcome").
5. Insert `[PLACEHOLDER]` tags where specific data or client approval is needed before submission.

## Output Format

Return the drafted content as a markdown document mirroring the template structure, with each section clearly headed.
