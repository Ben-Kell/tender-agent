# Controlled change and release governance model

Fujitsu applies a structured, Defence-controlled change and release governance model to ensure all modifications are delivered in a controlled and auditable manner.

All changes:
- Are raised in the Defence ITSM platform
- Include impact, risk, and dependency assessments
- Require Defence approval prior to execution

Changes are classified based on risk and complexity, which determines:
- Approval authority
- Level of analysis required
- Release scheduling

Release activities are:
- Planned into approved release windows
- Executed through controlled deployment processes
- Verified through post-release validation

This ensures:
- Defence retains full decision authority
- All changes are visible and traceable
- Operational stability is protected

This governance model supports controlled evolution of systems while maintaining service integrity.
